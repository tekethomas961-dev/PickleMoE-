# 匹克球四类击球分类的稀疏 MoE 模型实验报告

## 1. 任务目标与模型结构

本实验将单次匹克球击球识别建模为表格型四分类任务。输入为 8 个连续技术特征：`stance_x`、`stance_y`、`swing_speed`、`ball_speed`、`landing_x`、`landing_y`、`angle`、`height`；输出为 4 类击球标签：0 正手抽球、1 反手削球、2 网前吊球、3 高压球。

稀疏 MoE 模型由共享输入、门控网络、专家网络、Top-K 路由和稀疏加权组合构成：

```text
输入特征 x
  -> Gate 门控网络输出 4 个专家概率
  -> 选择 Top-K 专家，K=1 或 K=2
  -> 只运行被选中的专家
  -> 每个专家输出 4 类 logits
  -> 按 gate 权重加权融合 logits
  -> 输出最终 4 类预测
```

K=1 表示每个样本只激活 1 个专家，稀疏性最强，专家分工更清晰；K=2 表示每个样本激活 2 个专家并加权融合，表达能力更强，预测更平滑。

关键参数如下：

| 参数 | 取值 |
|---|---:|
| 输入维度 | 8 |
| 输出类别 | 4 |
| 专家数量 | 4 |
| 专家隐藏层 | 64 |
| Gate 隐藏层 | 32 |
| Top-K | 1 和 2 |
| Batch size | 64 |
| Epoch 上限 | 120 |
| 学习率 | 0.001 |
| 优化器 | Adam |
| 损失函数 | CrossEntropyLoss + lambda_lb * load balance loss |

## 2. 数据与训练设置

由于没有提供真实数据，本实验使用模拟数据。模拟依据是不同击球类型在速度、站位、落点、角度和高度上的技术差异：正手抽球和高压球通常挥拍速度、球速较高；网前吊球通常靠近网前、球速较低、落点较浅；反手削球角度更偏负向，速度中等。

数据集保持四类均衡，固定随机种子 `seed=42`。连续特征先按 `7:1:2` 划分为训练集、验证集、测试集，再仅用训练集拟合中位数填补器和标准化器，避免信息泄漏。

实验共比较 5 组模型：

| 模型 | 目的 |
|---|---|
| Dense MLP | 普通稠密基线，所有样本共享同一套参数 |
| MoE-K=1, lambda_lb=0.01 | 稀疏性最强，观察清晰专家分工 |
| MoE-K=2, lambda_lb=0.01 | 两专家加权融合，观察更稳定的组合预测 |
| MoE-K=1, lambda_lb=0 | 负载平衡消融实验 |
| MoE-K=2, lambda_lb=0 | 负载平衡消融实验 |

## 3. 实验结果

完整结果见 `runs/summary.md`。核心结果如下：

|---|---:|---:|---:|---:|---:|---:|
| Dense MLP | 63 | 1.000 | 1.000 | 1.000 | - | - |
| MoE-K=1, lb=0.01 | 42 | 1.000 | 1.000 | 1.000 | 0.174 | 1.125 |
| MoE-K=2, lb=0.01 | 67 | 1.000 | 1.000 | 1.000 | 0.163 | 1.238 |

## 4. 分析与总结

K=1 的特点是每个样本只由一个专家主导，因此专家分工更容易解释。例如在 K=1 结果中，正手抽球主要由 Expert 3 处理，网前吊球主要由 Expert 2 处理，高压球主要由 Expert 0 处理。反手削球在 Expert 1 和 Expert 2 之间分配，说明它可能处在两个技术模式之间。

K=2 允许两个专家共同参与预测，理论上边界样本会更稳定。本实验中，K=2 的 selected gate 权重反而呈现更明显的类别主导关系：正手抽球偏 Expert 3，反手削球偏 Expert 2，网前吊球偏 Expert 0，高压球偏 Expert 1。这说明 Top-2 路由在该模拟数据上没有削弱专家分工，反而利用第二专家提高了表示稳定性。

负载平衡损失是必要的加分项。无负载平衡时，K=1 的 `CV(load)` 为 0.777，`Max/Mean` 为 2.150，说明专家使用明显不均衡；加入 `lambda_lb=0.01` 后，K=1 的 `CV(load)` 降至 0.174，专家负载更稳定。K=2 也有类似现象，`CV(load)` 从 0.340 降至 0.163。由此可见，负载平衡损失能缓解专家坍缩和专家闲置，但系数不宜过大，否则可能强行平均分配样本，削弱门控网络的自主选择能力。

相比普通 MLP，MoE 的优势在于能够根据输入特征动态选择专家，更适合处理正手抽球、反手削球、网前吊球、高压球这类具有不同技术模式的数据。其不足是结构更复杂，需要额外分析 gate usage、负载平衡和专家分工；在小数据集上，MoE 不一定比强 MLP baseline 有更高准确率。因此本实验保留 Dense MLP 作为基线，并把 MoE 的重点放在稀疏路由机制和可解释专家分工上。
